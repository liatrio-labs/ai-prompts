# Skill Authoring Checklist

Use this checklist before finalizing a newly created skill.

- [ ] Skill has one coherent purpose
- [ ] Description states what it does and when to use it
- [ ] Clarification-first behavior is required for ambiguous requests
- [ ] Mode selection logic is explicit
- [ ] Workflow steps are imperative and ordered
- [ ] Inputs and outputs are explicit for each workflow mode
- [ ] Safety boundaries are explicit
- [ ] High-impact operations require approval
- [ ] Vendor-neutral output is the default
- [ ] Vendor-specific guidance is isolated in adapter references
- [ ] References are direct and shallow (no deep nesting)
- [ ] Templates are included for deterministic outputs
- [ ] Audit severity model is defined
- [ ] Markdown lint passes
- [ ] Any assumptions are explicitly documented
